# Mapping Systeme Meow Paw Paw

- `common` -> routeur minimal, tri des sous-cas, matrice de lecture, STOP de premier niveau.
- `update` -> runtime dense differe pour maintenance lourde, write-intent, contrats et change management.
- `sync` -> module cible pour nommage, ids, conventions de fichiers et alignement technique.
- `assets/` -> seules bases structurelles locales passives.
- `scripts/` -> controles deterministes repetables.

## Lecture courante
- toujours: `common`
- maintenance lourde / write-intent / audit dense: `update` puis `references/usage-update-runtime.md` si le detail operatoire est utile
- sync / nomenclature / ids / conventions: `sync`
- conditionnel: `sources-externes`
- migration uniquement: `usage-migration`
