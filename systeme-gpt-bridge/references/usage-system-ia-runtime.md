# Usage system ia runtime

`System IA` est le sous-cas le plus sensible de ce skill parce qu il melange :

- prompt projet
- index
- journal
- ecosystem context
- copies d instructions et de chemins
- traces de l ancien modele `MARWAN` / `BEBERT`

Lecture minimale :

1. `governance/common.regles.json`
2. `governance/projects.regles.json`
3. `governance/system-ia.regles.json`

Lire ensuite seulement les fichiers reels qui deviennent necessaires dans :

- `C:/Users/engue/OneDrive/__SYSTEME - PERSONA_O.S__/GPT/System IA/System IA ATTACHEMENTS`

Principes :

- ne pas supposer qu un fichier copie est encore canonique
- traiter les copies `PROMPT__*` et `INDEX__*` comme artefacts de migration tant qu elles ne sont pas realignees
- ne pas rebrancher de nouvelles dependances vers `MARWAN.agent.md` ou `BEBERT.agent.md`
- si un changement touche le routeur global local, verifier la coherence avec `C:/Users/engue/.codex/AGENTS.md`

Quand stopper :

- si un fichier legacy semble encore actif mais que son remplacant cible n est pas arbitre
- si plusieurs copies concurrentes pretendues canoniques existent
- si la demande melange maintenance GPT et mutation metier hors perimetre
