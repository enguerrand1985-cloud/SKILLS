# Update Light

Utiliser ce chemin pour :
- petite correction de `SKILL.md`
- ajout isole d'asset
- ajout isole de script
- ajout isole de reference
- petite mise a jour de metadata

## Regles
- ne lire que le module concerne
- ne pas relire toute la norme canonique si le cas reste local
- ne pas escalader tant qu'aucun changement structurel n'apparait
- conserver la separation entre `governance`, `references`, `scripts` et `assets`

## Escalade
Passer au chemin intermediaire si :
- l'ordre de lecture change
- la structure des dossiers change
- la gouvernance doit etre introduite ou recomposee

Passer au chemin complet si :
- la migration devient large
- le skill devient modulaire ou hybride
- un schema lourd doit etre revalide
