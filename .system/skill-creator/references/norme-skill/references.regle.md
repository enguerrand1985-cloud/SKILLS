# References

`references/` contient du support conditionnel :
- mapping
- intake
- usage detaille
- exemples
- sources externes

## Formats canoniques
- `mapping-<skill>.md`
- `intake-<skill>.md`
- `usage-<module>.md`
- `examples-<module>.md`
- `sources-externes.md`

## Regles
- garder la logique active dans `governance/`
- utiliser `references/` pour la doctrine detaillee et les procedures longues
- ne pas transformer `references/` en verite locale
- ne pas remplacer un contrat JSON actif par un simple Markdown
